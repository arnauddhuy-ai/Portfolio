TP : Centralisation et Sécurisation d'une Infrastructure PME

1. Introduction

Dans une PME de 100 postes, l’entreprise souhaite centraliser la gestion des utilisateurs, des ressources et de la sécurité via un serveur Windows Server 2022 configuré en contrôleur de domaine.

Ce TP vous permettra de :

  • Compléter et configurer l’infrastructure Active Directory existante
  • Déployer et tester DHCP et DNS
  • Organiser les OU et groupes de sécurité
  • Créer des partages réseau avec permissions combinées (NTFS + partage)
  • Appliquer des GPO pour sécuriser et personnaliser les postes clients
  • Vérifier et valider toute l’infrastructure

Serveur de référence : SRV-AD | Domaine : entreprise.local | IP : 192.168.77.10/24

2. Objectifs pédagogiques

Après ce TP, vous serez capable de :

  • Vérifier et configurer la connectivité réseau des serveurs et postes clients
  • Installer et configurer les rôles DHCP et DNS sur Windows Server 2022
  • Organiser et structurer un domaine Active Directory avec OU, utilisateurs et groupes de sécurité
  • Créer des partages réseau avec permissions combinées NTFS et partage
  • Appliquer des GPO ciblées pour sécuriser et personnaliser les postes clients
  • Tester et valider le fonctionnement des services et des politiques appliquées
  • Comprendre l’importance de la centralisation et de la sécurité dans une infrastructure d’entreprise

3. Vérification de la configuration réseau de la VM

3.1 Dans VMware

Sélectionnez votre VM Windows Server (SRV-AD) > Menu : VM > Settings > Network Adapter.

ModeUsage pour le TP
BridgedVM sur le réseau physique réel
NATVM partage l’IP de l’hôte pour Internet
Host-onlyVM isolée mais communique avec hôte et autres VMs (Choix TP)

3.2 Vérification réseau côté serveur

CMD :

ipconfig /all

Vérifiez que le serveur utilise son IP comme DNS (192.168.77.10).

4. Configuration de l’adresse IP fixe du serveur

  • IP : 192.168.77.10
  • Masque : 255.255.255.0
  • Passerelle : 192.168.77.1 (fictive en host-only)
  • DNS : 192.168.77.10

Vérification CMD :

ipconfig /all
ping 192.168.77.10
Astuce : Tester la connectivité avec un poste client avant de continuer.

5. Installation et configuration du rôle DHCP

5.1 Installation du rôle

Gestionnaire de serveur > Ajouter rôles et fonctionnalités > DHCP Server > Installer. Autorisez DHCP dans Active Directory.

5.2 Création de l’étendue DHCP

IPv4 > Nouvelle étendue :

  • Nom : Etendue_VM
  • Plage IP : 192.168.77.100 → 192.168.77.200
  • Masque : 255.255.255.0
  • Passerelle (003) : 192.168.77.1
  • DNS préféré (006) : 192.168.77.10

5.3 Réservation DHCP

MAC : 00:11:22:33:44:55 | IP réservée : 192.168.77.150

5.4 Vérification côté client

ipconfig /release
ipconfig /renew
ipconfig /all
ping 192.168.77.10

8.1 Intégration Client au Domaine

8.3 GPO Comptabilité_Politique

  • Panneau de configuration : Activé (Interdire l'accès).
  • Redirection Documents : De base vers \\SRV-AD\Compta (créer un dossier pour chaque utilisateur).

8.4 GPO Stagiaires_Politique

  • Bureau : Masquer "Mes documents", "Internet Explorer", "Emplacements réseau".
  • Fond d'écran : Activer avec chemin \\SRV-AD\Public\wallpaper_ok.jpg.

9. Dépannage Rapide

  • GPO non appliquée : vérifiez gpresult /r, DNS client, liaison GPO.
  • Redirection non créée : vérifiez droits NTFS & partage.
  • Fond d'écran : vérifiez chemin UNC accessible.
reg query "HKCU\Control Panel\Desktop" /v Wallpaper

10. Questions de validation

Différence groupe Sécurité / Distribution : Sécurité pour les droits d'accès, Distribution pour les listes d'emails.

NTFS vs Partage : NTFS s'applique localement et réseau, Partage uniquement réseau. Le plus restrictif gagne.

Priorité GPO : LSDOU (Local, Site, Domaine, Unité d'Organisation).

Conclusion

Ce TP a permis de mettre en place une infrastructure Windows Server 2022 complète pour une PME, en centralisant la gestion des utilisateurs, des ressources et de la sécurité.
